Pattern: Custom Accordion Component

Problem

Foundation 5 no longer supports Sections, which means that both Tabs and Accordions will not work. The long term solution is to use the new Tabs and Accordion components native to Foundation 5 and 6. Because much of the legacy CSS / Script target elements managed by a specific version of the framework, upgrading would require significant code changes.

Solution

This is a custom bridge implementation of the Foundation 4 Accordion (AKA Sections) that works without F4.

moveSecTab();
$(document).ajaxComplete(function() {
    moveSecTab();
});
$(document).on("opened.fndtn.reveal", "[data-reveal]", function () {
    var oTOtabs = window.setTimeout(moveSecTab, 200);
});
$(document).on("click", ".section-container .title>a", function (e) {
    e.preventDefault();  
    moveSecTab();
    $(".section-container section.active").removeClass("active");
    $(this).closest("section").addClass("active");
    var oTOtabs = window.setTimeout(moveSecTab, 200);
});
function moveSecTab(){
    if ($(".section-container")[0]){
        var sWi = parseInt( $(".section-container.tabs section:nth-child(1n) .title").css("width"), 10);
        $(".section-container.tabs section:nth-child(2n) .title").css("left", (sWi-3)+"px");
    }
}
