Pattern: Skip Navigation (A11y)

Problem

The main content is not usually the first thing on a web page. Keyboard and screen reader users generally must navigate a long list of navigation links, sub-lists of links, corporate icons, site searches, and other elements before ever arriving at the main content. This is particularly difficult for users with some forms of motor disabilities. Without some sort of system for bypassing the long list of links, some users are at a huge disadvantage. Consider users with no arm movement, who use computers by tapping their heads on a switch or that use a stick in their mouth to press keyboard keys. Requiring users to perform any action perhaps 100s of times before reaching the main content is simply unacceptable.

Solution

provide a link at the top of the page which jumps the user down to an anchor or target at the beginning of the main content.

Reference

Using ARIA landmarks to identify regions of a page 
<a id="js-acs-skip--a" class="acs-skip--a resp_u"
        href="#acs-skip__target"
        tabindex=1>Skip to main content</a>

function setAcs_skip(){
    //    Create Skip to main conent link target
    //    Only if large viewport AND two column layout
    if(jsMQ.isLarge() && ($(".large-8").length>=0) && ($("#js-acs-skip--a").length>=0) ){
        $(".large-8").first().attr("id", "acs-skip__target");
    }else{
        $("#js-acs-skip--a").remove();
    }
}

.acs-skip--a { 
position:absolute; 
left:-10000px; 
top:auto; 
width:1px; 
height:1px; 
overflow:hidden;
font-size: 500;
} 
 
.acs-skip--a:focus { 
top: 8px;
left: 22px;
width:auto; 
height:auto; 
}
